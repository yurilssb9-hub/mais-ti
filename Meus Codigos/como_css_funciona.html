Sintaxe básica — como uma regra CSS funciona

CSS funciona por regras. Cada regra diz quem recebe estilo (seletor) e o quê muda (declarações).

/* Estrutura de uma regra CSS */ seletor { propriedade: valor; propriedade: valor; } /* Exemplo real */ p { color: #333333; font-size: 16px; line-height: 1.6; }

🎯 Seletor

Define quais elementos HTML serão afetados. Pode ser uma tag, uma classe, um ID ou combinações.

🔧 Propriedade

O aspecto visual que será alterado: color, font-size, margin, background…

✨ Valor

Como a propriedade será aplicada: red, 16px, bold, #fff, 1fr…

Três formas de aplicar CSS

1️⃣ Inline

Direto no atributo style do elemento. Prático para testes, péssimo para manutenção.

<p style="color:red;"> Texto </p>

2️⃣ Interno (<style>)

Dentro de uma tag <style> no <head>. Útil para páginas únicas ou prototipagem.

<style> p { color: blue; } </style>

3️⃣ Externo ✅

Arquivo .css separado, linkado pelo <head>. Melhor prática para qualquer projeto real.

<link rel="stylesheet" href="styles.css" />
🏗️ Por que arquivo externo é melhor? Um único arquivo styles.css pode ser compartilhado por dezenas de páginas HTML. Se você quiser mudar uma cor, muda em um único lugar — e todas as páginas atualizam automaticamente.

Seletores — quem recebe o estilo?

O seletor é o "endereço" do elemento. Quanto mais específico, maior a prioridade (veremos isso na Cascata).

/* Seletor de ELEMENTO — afeta todos os p */ p { color: #333; } /* Seletor de CLASSE — afeta qualquer .destaque */ .destaque { background: yellow; } /* Seletor de ID — afeta apenas #unico */ #unico { color: red; } /* DESCENDENTE — p dentro de qualquer nav */ nav p { font-size: 0.9rem; } /* FILHO DIRETO — somente li filho direto de ul */ ul > li { list-style: none; } /* PSEUDO-CLASSE — estado especial */ a:hover { color: blue; } li:first-child { font-weight: bold; } /* MÚLTIPLOS — aplica o mesmo estilo a vários */ h1, h2, h3 { font-family: serif; }

Box Model — todo elemento é uma caixa

Este é o conceito mais importante do CSS para layout. Cada elemento HTML é tratado como uma caixa retangular com 4 camadas.

margem (margin)
borda (border)
padding
conteúdo (content)

📏 Código do Box Model

.caixa { /* Dimensão do conteúdo */ width: 300px; height: 150px; /* Espaço interno */ padding: 20px; /* Borda */ border: 2px solid #333; /* Espaço externo */ margin: 16px auto; }

🟦 margin vs padding

Padding é o espaço entre o conteúdo e a borda — está dentro da caixa. Clicável e colorível.

Margin é o espaço entre a borda e os elementos vizinhos — está fora da caixa. Transparente sempre.

💡 box-sizing: border-box
Por padrão, width conta só o conteúdo. Com box-sizing: border-box, padding e borda entram na conta do width — muito mais intuitivo! Por isso quase todo CSS moderno começa com:

* { box-sizing: border-box; }

⚡ Atalhos de margin e padding

/* 1 valor — todos os lados iguais */ margin: 16px; /* 2 valores — vertical | horizontal */ padding: 10px 20px; /* 4 valores — topo | direita | baixo | esquerda (sentido horário) */ margin: 10px 20px 30px 0; /* Lados individuais */ margin-top: 24px; padding-left: 12px; /* Centralizar horizontalmente (precisa de width definida) */ margin: 0 auto;

Cascata e especificidade — quem vence o conflito?

Quando duas regras CSS tentam estilizar o mesmo elemento da mesma propriedade, uma delas vence. A ordem é:

MAIOR PESO Inline style — estilo direto no HTML: style="..."
ALTO Seletor de ID — #meu-id { ... }
MÉDIO Seletor de Classe / Pseudo-classe — .minha-classe { ... } / :hover
MENOR Seletor de Elemento — p { ... } / h2 { ... }
🤔 Qual cor vai aparecer?
p { color: blue; /* elemento */ } .texto { color: green; /* classe */ } #titulo { color: red; /* id */ } <!-- HTML --> <p class="texto" id="titulo"> Qual cor? </p>
✅ Resposta
/* O ID tem maior especificidade. */ /* Resultado: texto VERMELHO */ /* Se o elemento tivesse só class: */ /* → VERDE (classe vence elemento) */ /* Se tivesse só a tag p: */ /* → AZUL (somente o elemento) */ /* Inline style sempre vence todos: */ <p style="color:orange"> /* → LARANJA, independente do CSS */
⚠️ Sobre o !important: A declaração color: red !important; força um valor a vencer qualquer outra regra. É a arma nuclear do CSS — funciona, mas cria um caos de manutenção. Evite a não ser em casos muito específicos.

Variáveis CSS (Custom Properties)

Variáveis CSS permitem definir valores uma vez e reutilizar em todo o arquivo. Mudar uma cor no projeto inteiro vira uma edição de uma linha.

❌ Sem variáveis
header { background: #e34c26; } button { background: #e34c26; } a:hover { color: #e34c26; } /* Mudar a cor = editar 3 lugares */
✅ Com variáveis
:root { --cor-primaria: #e34c26; } header { background: var(--cor-primaria); } button { background: var(--cor-primaria); } /* Mudar a cor = editar 1 lugar */
💡 Por que :root? O :root é o elemento mais alto da árvore HTML — equivale ao html. Variáveis definidas ali ficam disponíveis em qualquer elemento da página (herança CSS). É a convenção padrão da indústria.

🎨 Formatos de cor no CSS

Todos esses formatos representam cores válidas:

#e34c26 (hex)
rgb(227,76,38)
rgba(..., 0.5) (com transparência)
hsl(15, 75%, 52%)
tomato (nome CSS)
transparent
💡 O CSS possui 148 nomes de cor reconhecidos! De aliceblue a yellowgreen. São convenientes para protótipos, mas use hex ou hsl em projetos reais para controle preciso.

Flexbox e Grid — organização de layout

Antes de 2015, layout era feito com float e posicionamentos manuais — um pesadelo. Hoje temos duas ferramentas poderosas:

↔️ Flexbox — uma dimensão
/* No elemento PAI */ .menu { display: flex; gap: 16px; /* Direção */ flex-direction: row; /* →→→ */ flex-direction: column; /* ↓↓↓ */ /* Alinhamento */ justify-content: center; align-items: center; /* Quebra de linha */ flex-wrap: wrap; }
⊞ Grid — duas dimensões
/* No elemento PAI */ .cards { display: grid; gap: 24px; /* 3 colunas de tamanho igual */ grid-template-columns: repeat(3, 1fr); /* Responsivo sem media query! */ grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); }

🤔 Quando usar Flexbox vs Grid?

SituaçãoUse
Links de navegação lado a ladoFlexbox
Cards de produtos em gradeGrid
Centralizar um elemento (vertical + horizontal)Flexbox
Layout de página completo (header, sidebar, main)Grid
Itens de tamanho variado em linhaFlexbox
Galeria de imagens alinhadasGrid
Botões de formulário lado a ladoFlexbox
📱 Responsividade com media query: Você pode mudar o layout em telas menores usando @media. Exemplo: @media (max-width: 600px) { .cards { grid-template-columns: 1fr; } } — isso faz a grade virar uma coluna só em celulares.

Curiosidades e boas práticas

📐 px vs rem vs em vs %

Todas são unidades de medida, mas se comportam diferente:

UnidadeRelativa aUso típico
pxPixel fixoBordas, sombras
remFonte do <html>Fontes, espaçamentos
emFonte do paiPadding de componentes
%Elemento paiLarguras fluidas
vw/vhViewportSeções de tela cheia

🔄 A propriedade transition

Anima mudanças de estado (como hover) suavemente — sem JavaScript!

button { background: blue; /* transição de 0.2s */ transition: background 0.2s; } button:hover { background: darkblue; } /* O navegador anima sozinho! */

🧹 Reset vs Normalize

Cada navegador tem estilos padrão diferentes. Para começar do zero, muitos projetos usam um CSS reset:

/* Reset mínimo — base de quase todo projeto */ * { margin: 0; padding: 0; box-sizing: border-box; }

🖨️ CSS para impressão

Você pode definir estilos só para quando a página for impressa:

@media print { nav, footer, button { display: none; } body { font-size: 12pt; color: black; } }

📦 display: none vs visibility: hidden vs opacity: 0

Três formas de "esconder" um elemento — com comportamentos muito diferentes:

DeclaraçãoVisível?Ocupa espaço?Clicável?
display: none❌ Não❌ Não — removido do fluxo❌ Não
visibility: hidden❌ Não✅ Sim — espaço mantido❌ Não
opacity: 0❌ Não✅ Sim — espaço mantido✅ Sim!
🎭 opacity: 0 + pointer-events: none = invisível e não clicável, mas ainda ocupa espaço. Muito usado em animações de fade-in.
