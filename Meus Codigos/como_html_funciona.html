1

Estrutura base de um documento HTML

Todo arquivo HTML começa com a mesma estrutura mínima. Cada parte tem uma função específica — nenhuma é decoração.

<!-- Declara o tipo de documento (sempre a 1ª linha) --> <!DOCTYPE html> <!-- Define o idioma da página (importante para SEO e acessibilidade) --> <html lang="pt-BR"> <head> <!-- Diz ao navegador para usar UTF-8 (acentos funcionam!) --> <meta charset="UTF-8" /> <!-- Faz a página funcionar bem em celulares --> <meta name="viewport" content="width=device-width, initial-scale=1.0" /> <!-- Título que aparece na aba do navegador --> <title>Minha Página</title> <!-- Liga o arquivo CSS externo --> <link rel="stylesheet" href="styles.css" /> </head> <body> <!-- Todo conteúdo visível da página fica aqui --> ... </body> </html>
🌐 Curiosidade: O <!DOCTYPE html> não é uma tag HTML — é uma instrução para o navegador. Sem ela, alguns navegadores entram em modo de compatibilidade antiga e a página pode se comportar de forma inesperada.

🧠 <head> — o cérebro

Invisível para o usuário. Contém configurações, links para CSS, metadados para SEO e o título da aba.

👁️ <body> — o corpo

Tudo que o usuário vê: textos, imagens, botões, formulários, menus. Só o que está aqui aparece na tela.

2

Tags semânticas — significado além da aparência

Uma tag semântica comunica o papel do conteúdo — não apenas como ele deve parecer. O navegador, o Google e leitores de tela entendem a diferença.

❌ Sem semântica (div soup)
<div id="topo"> <div id="nav">...</div> </div> <div id="conteudo"> <div class="post">...</div> </div> <div id="rodape">...</div>
✅ Com semântica (HTML5)
<header> <nav>...</nav> </header> <main> <article>...</article> </main> <footer>...</footer>
💡 Dica: Ambos os códigos acima podem ter a mesma aparência na tela. A diferença é que o segundo comunica intenção — e isso importa para SEO, acessibilidade e para outros desenvolvedores lerem seu código.
Tag O que representa Quando usar
<header> Cabeçalho da página ou de uma seção Logo, título, navegação principal
<nav> Bloco de navegação Menus, links de seções, breadcrumbs
<main> Conteúdo principal e único da página Deve existir apenas uma vez por página
<section> Seção temática de conteúdo Agrupa conteúdos relacionados (com título)
<article> Conteúdo independente e autocontido Posts, pratos do menu, cards de produto
<aside> Conteúdo complementar Barras laterais, notas, anúncios relacionados
<footer> Rodapé da página ou seção Copyright, links institucionais, contato
<figure> / <figcaption> Imagem com legenda Fotos, gráficos, ilustrações explicativas
<time> Data ou hora Publicações, eventos, horários
<address> Informações de contato Endereço, e-mail, telefone do autor/empresa
📚 Curiosidade: A tag <article> surgiu para descrever conteúdo que faz sentido por si só — como um post de blog. Você pode copiar um <article> e colocá-lo em outra página que ele ainda fará sentido. Isso não vale para uma <section>.
3

Hierarquia, indentação e a Árvore DOM

O HTML organiza elementos em uma árvore: todo elemento tem um pai, pode ter filhos, e filhos no mesmo nível são chamados de irmãos.

📝 Código HTML

<body> <header> <h1>Título</h1> <nav> <ul> <li><a>Link</a></li> </ul> </nav> </header> <main> <section> <h2>Seção</h2> <p>Texto</p> </section> </main> </body>

🌳 Árvore DOM gerada

body
├── header
│ ├── h1
│ └── nav
│ └── ul
│ └── li → a
└── main
└── section
├── h2
└── p
💡 Regra de ouro da indentação: Cada elemento filho deve ter 2 espaços (ou 1 tab) a mais que o elemento pai. Código bem indentado não é perfumaria — é profissionalismo.

📐 A hierarquia dos títulos: h1 ao h6

Os títulos formam um outline da página, como capítulos de um livro. Nunca pule um nível só por questão visual — use CSS para tamanho.

<h1>Título principal da página (único!)</h1> <h2>Seção principal</h2> <h3>Subseção</h3> <h4>Sub-subseção (raramente necessário)</h4>
⚠️ Atenção: Use apenas um <h1> por página. Pular do h1 para h3 (sem h2) é um erro semântico — o Google interpreta isso como estrutura quebrada.
4

Atributos id e class — identificando elementos

São os dois atributos mais usados para conectar HTML com CSS e JavaScript.

🔑 id — identidade única
<!-- Só pode aparecer UMA VEZ na página --> <section id="contato">...</section> <!-- Link âncora usa # + id --> <a href="#contato">Ir ao contato</a> <!-- CSS: seletor com cerquilha --> #contato { background: #f5f5f5; }
🏷️ class — rótulo reutilizável
<!-- Pode ser usado em VÁRIOS elementos --> <article class="card">...</article> <article class="card">...</article> <!-- Um elemento pode ter VÁRIAS classes --> <p class="texto destaque grande">...</p> <!-- CSS: seletor com ponto --> .card { border: 1px solid #ccc; }
🤔 Por que id e class existem separados? O id nasceu para identificação única (como um CPF). O class é como uma categoria (como "produto em promoção"). Um produto pode estar em várias categorias, mas só tem um CPF — a mesma lógica vale aqui.
5

Formulários — coletando dados do usuário

Formulários são feitos com várias tags em conjunto. A boa semântica aqui melhora acessibilidade: leitores de tela conseguem associar cada campo ao seu rótulo.

<form> <!-- label + input ligados pelo for/id --> <label for="nome">Nome</label> <input type="text" id="nome" name="nome" placeholder="Seu nome" /> <!-- type="email" valida o formato automaticamente --> <label for="email">E-mail</label> <input type="email" id="email" name="email" /> <!-- textarea: campo de texto longo --> <label for="msg">Mensagem</label> <textarea id="msg" name="msg" rows="4"></textarea> <!-- select: lista de opções --> <label for="assunto">Assunto</label> <select id="assunto" name="assunto"> <option value="reserva">Reserva de mesa</option> <option value="duvida">Dúvida</option> </select> <button type="submit">Enviar</button> </form>
type do <input>Para que serveValidação automática
textTexto livreNenhuma
emailEndereço de e-mailVerifica se tem @
passwordSenha (texto ocultado)Nenhuma
numberNúmerosAceita só dígitos
dateData (calendário)Formato de data
checkboxCaixa de marcaçãoNenhuma
radioEscolha única entre opçõesNenhuma

📱 Curiosidade: O type="email", type="tel" e type="number" abrem teclados diferentes no celular — o navegador escolhe o teclado certo automaticamente. Isso é comportamento semântico puro: a tag diz o que o dado significa, e o navegador age em conformidade.
6

Curiosidades e boas práticas

♿ Acessibilidade com alt

O atributo alt em imagens é lido por leitores de tela para pessoas com deficiência visual. Deixá-lo vazio (alt="") diz ao leitor "ignore esta imagem". Omiti-lo é um erro.

<!-- ✅ Correto --> <img src="logo.png" alt="Logo da Esquina do Sabor" /> <!-- ❌ Errado (sem alt) --> <img src="logo.png" />

🔗 Links e o atributo href

O href="#" é um placeholder — o link não vai a lugar nenhum. Use âncoras reais (#secao) ou URLs completas.

<!-- Link externo: abre em nova aba --> <a href="https://google.com" target="_blank" rel="noopener">Google</a> <!-- Âncora interna --> <a href="#contato">Ir ao contato</a>

📦 <div> vs tags semânticas

A <div> ainda existe e é útil — mas apenas quando nenhuma tag semântica se encaixa. Ela é um container genérico para fins de CSS/JS.

💡 Regra prática: antes de usar <div>, pergunte "existe uma tag com significado para isso?" Se sim, use ela.

🏷️ Atributo lang no <html>

Definir lang="pt-BR" no <html> tem impacto real: leitores de tela usam a pronúncia correta, e o Chrome pergunta se deve traduzir a página.

🌍 Existem mais de 8000 códigos de idioma no padrão BCP 47. pt-BR é português do Brasil, pt-PT é de Portugal, en-US é inglês americano.
